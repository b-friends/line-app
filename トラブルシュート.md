# トラブルシュート

## 2025-01 LIFF SDK が Apps Script ウェブアプリ内で動作しない問題

### 症状

- LIFF URL (`https://liff.line.me/xxxx`) を開くと「LIFF_ID が未設定です。」または初期化失敗になる
- PC ブラウザの Console に以下のエラーが表示される:
  - `liff.init() was called with a current URL that is not related to the endpoint URL`
  - `Refused to display 'https://access.line.me/' in a frame because it set 'X-Frame-Options' to 'deny'`

### 原因

Google Apps Script の HTML Service は、HTMLを `googleusercontent.com` ドメインの **iframe 内** で表示する。

1. **URL不一致**: LIFF のエンドポイントURLは `script.google.com/macros/s/.../exec` だが、実際のページは `n-xxx.googleusercontent.com/userCodeAppPanel` で表示される。LIFF SDK は現在のURLがエンドポイントURLと一致しないため初期化に失敗する。

2. **iframe制約**: LINE のログイン画面 (`access.line.me`) は `X-Frame-Options: deny` を設定しており、iframe 内で開くことができない。Apps Script の HTML は常に iframe 内なので、LINE認証画面を表示できない。

3. `setXFrameOptionsMode(ALLOWALL)` を設定しても、Apps Script 側の iframe は解除できるが、LINE 側の iframe 拒否は解決できない。

### 試したこと（効果なし）

- `setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL)` 追加
- テンプレートタグ `<?=` → `<?!=` への変更（エスケープ問題の切り分け）
- LIFF SDK のバージョン確認
- LINE Login チャネルの「公開」設定

### 解決策

**Apps Script をフロントとして使うのをやめ、API モードに変更する。**

- **フロント**: GitHub Pages（静的HTML）に LIFF SDK を含む画面を配置
- **バックエンド**: Apps Script は `doGet()`/`doPost()` で JSON を返す API として機能
- **LIFF エンドポイント**: GitHub Pages の URL を設定

GitHub Pages は独自ドメインで iframe なしに表示されるため、LIFF SDK が正常に動作する。

### 構成変更

| | v1 (webapp) | v2 (API + GitHub Pages) |
|---|---|---|
| フロント | Apps Script HTML Service | GitHub Pages |
| バックエンド | Apps Script (doGet で HTML返却) | Apps Script (doGet/doPost で JSON返却) |
| LIFF エンドポイント | script.google.com URL | GitHub Pages URL |
| 認証フロー | iframe 内で失敗 | 正常動作 |

- [Apps Script HTML Service の制約](https://developers.google.com/apps-script/guides/html/restrictions)
